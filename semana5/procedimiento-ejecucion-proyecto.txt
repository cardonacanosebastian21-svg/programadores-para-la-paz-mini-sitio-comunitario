Escribe el procedimiento para que otra persona pueda ejecutar el proyecto.

1. Requisitos del entorno

¿Qué debe tener instalado la persona?
Ejemplos: Git, Node.js, navegador, editor de código.

Respuesta: Debe tener instalado Git (para control de versiones), Node.js (entorno de ejecución para el backend), un editor de código (como Visual Studio Code) y un navegador web moderno.

2. Clonar repositorio

Comando: git clone https://github.com

Respuesta: Este comando descarga una copia exacta del código del proyecto desde GitHub a la computadora local.

3. Entrar a la carpeta del proyecto

Comando: cd 

Respuesta: Cambia el directorio de trabajo de la terminal para situarse dentro de la carpeta raíz del proyecto.

4. Crear o cambiar a rama personal

Comando: git checkout

Respuesta: Cambia el entorno de trabajo a tu rama personal para asegurar que los cambios se guarden en tu espacio asignado.

5. Instalar dependencias

Comando: npm install

Respuesta: Descarga e instala todas las librerías necesarias (como Express para la API) configuradas en el archivo package.json.

6. Ejecutar el proyecto

Comando: npm run sitio

Respuesta: Enciende el servidor local de Node.js y pone en funcionamiento el backend y las rutas de la API.

7. Abrir en navegador

Dirección: http://localhost:3000

Respuesta: Es la dirección de red local que permite visualizar e interactuar con la interfaz del frontend en el navegador.

8. Probar rutas de la API

Escribe las rutas que se deben probar: Las rutas que se deben probar son http://localhost:3000/api/mensajes, http://localhost:3000/api/calendario y http://localhost:3000/api/resumen.

Respuesta: Se probaron en el navegador.

9. Probar login pedagógico

Usuario de demostración: docente

Clave de demostración: demo

Resultado esperado: Se ingresa el usuario docente con la clave demo y el resultado esperado es una simulación de inicio de sesión exitoso con fines educativos.

10. Verificación final

¿Cómo sabe la persona que el proyecto está funcionando?

Respuesta: Se confirma el funcionamiento si la terminal no muestra errores y el navegador renderiza el tablero inyectando dinámicamente los mensajes JSON desde la API.

11. Nota de seguridad

¿Qué información NO debe publicarse?

Respuesta: No se deben publicar claves reales, tokens privados, datos personales de los vecinos ni credenciales del servidor en el código de acceso público.
