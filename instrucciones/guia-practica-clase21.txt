Guía práctica – Clase 21

Actividad:
Inicio del proyecto único de Semana 5: mini-sitio comunitario que consume una API con async y await.

Repositorio:
Repositorio dispuesto por el docente.

Importante:
Este repositorio se usará desde la Clase 21 hasta la Clase 25.

No debes trabajar directamente en main.

Cada estudiante debe crear una rama personal con el formato:

nombres_apellidos

Evidencia:
La evidencia de esta actividad es únicamente el push de tu rama personal a GitHub.

No debes entregar nada en Moodle.
No debes enviar capturas.
No debes subir archivos adicionales.
No debes publicar enlaces por otro medio.

Comando nuevo del día:
npm run sitio

Pasos obligatorios:

1. Clonar el repositorio

git clone URL_DEL_REPOSITORIO

2. Entrar a la carpeta del proyecto

cd programadores-para-la-paz-mini-sitio-comunitario

3. Crear la rama personal

git checkout -b nombres_apellidos

4. Verificar la rama

git branch

La rama marcada con * debe ser tu rama personal.

5. Identificar el flujo de actualización que se usará desde Clase 22

git pull origin main

6. Instalar dependencias

npm install

7. Levantar el mini-sitio

npm run sitio

8. Abrir en el navegador

http://localhost:3000

9. Probar botón Cargar mensajes

10. Probar botón Cargar calendario

11. Abrir ruta directa de mensajes

http://localhost:3000/api/mensajes

12. Abrir ruta directa de calendario

http://localhost:3000/api/calendario

13. Revisar public/app.js

Identificar:
async
await
fetch
try
catch
respuesta.json()

14. Modificar data/mensajes.json ajustando un mensaje

15. Modificar data/calendario-editorial.json ajustando una pieza del calendario

16. Detener servidor con Ctrl + C

17. Levantar nuevamente

npm run sitio

18. Probar nuevamente los botones

19. Completar archivos:

semana5/conceptos-async-await.txt
semana5/bitacora-git-proyecto-vivo.txt
semana5/pruebas-mini-sitio-api.txt
semana5/calendario-editorial-2-semanas.txt
semana5/checklist-difusion-responsable.txt
semana5/reflexion-mini-sitio-comunitario.txt

20. Revisar cambios

git status

21. Agregar cambios

git add .

22. Crear commit

git commit -m "Clase 21 inicio proyecto unico mini sitio"

23. Publicar rama

git push -u origin nombres_apellidos
